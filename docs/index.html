Autopilot — drop a resume, walk away

Lazy mode for serious job hunters. Drop a resume (or paste your LinkedIn), pick employment type + interview availability, click START AUTOPILOT. In ~5 seconds the skill:

  1. Infers your profile from the resume — name, contact, current title, skills, location, currency — and suggests your next-rung target titles based on trajectory (Senior → Staff, IC → Manager).
  2. Builds the Career Evidence Vault with provenance for every claim.
  3. Searches every enabled job board in parallel and dedupes.
  4. Scores every job against your evidence with plain-English reasons.
  5. Tailors a resume + cover letter for your top matches — never inventing facts.
  6. Builds application packets for the top picks, ready to review.
  7. Schedules a daily re-run so new postings flow in while you sleep.

You can switch to Custom Setup for fine-grained control. Either way, no application is submitted on your behalf — we prepare the packet and queue it for your review.

What is Job Hunt Hacker?

Think of it as a private, honest, tireless headhunter who lives on your laptop and works only for you. A great human recruiter does seven things well: search proactively, match against your real background, tailor materials per role, prepare you for interviews, negotiate offers, know the market, and maintain a network of warm intros. Job Hunt Hacker does all seven. It just doesn't take 25% of your first-year salary.

Everything runs on your machine. Your Career Evidence Vault (the canonical record of what you've actually done) is a local SQLite file at data/jhh.db. No cloud account. No telemetry. No subscription. Use any LLM you like — Anthropic, OpenAI, local Ollama — or use the deterministic template fallback for zero cloud egress.

Operate it through the brutalist single-page web UI at http://127.0.0.1:8731, through Claude Code, or via the REST API. The skill is MIT-licensed and works for any industry, not just tech.

What it does NOT do

Invent facts

Never. Every resume bullet, cover-letter paragraph, recruiter message, and talking point traces to ≥1 evidence ID. Anything that can't be tied to evidence is dropped to the gap report — not shipped.

Bypass platform protections

No CAPTCHA solving, no login bypass, no anti-detection fingerprinting. Scraped sources (LinkedIn, Indeed, Glassdoor) are GRAY-tagged; we prepare assisted-apply packets and you click through.

Auto-submit applications

Even when "auto-apply" is enabled, the skill only PREPARES packets and queues them for your review with status auto_packet_ready. Submission stays manual.

Send emails on your behalf

Drafts only. The skill writes recruiter messages, followups, and interview-availability replies — you read and send.

Edit your LinkedIn

We never log into LinkedIn for you. You paste your profile text (or HTML export) and we parse it locally.

Upload your data anywhere

The Vault lives in data/jhh.db on your machine. No telemetry. The only network egress is to the job boards and your chosen LLM provider.

How tailoring stays honest

Every generated artifact ships with a provenance map (each segment cites which EvidenceSource backs it) and an honesty report (counts of facts used, keywords added, keywords excluded as unsupported, gaps flagged). What that looks like:

TYPICAL AI RESUME TOOL - Led migration of 12-engineer team to microservices, reducing latency 47%. You have one resume bullet that says "worked on backend". The AI invented the team size, the architecture decision, and the percentage. Now you have to defend it in the interview.
JOB HUNT HACKER - Backend engineering on the orders service. Cites EvidenceSource #14 (your 2023 self-review). The honesty report flags "team size, architecture, latency metric: missing evidence — add or accept the gap."

If a posting requires Kubernetes and you have no Kubernetes evidence, the tool refuses to add it. It suggests either uploading a project as new evidence or accepting the gap. You stay credible; the recruiter gets reality.

Five-minute install

Requirements: Python 3.10+. macOS or Linux (Windows via WSL).

install.sh 
$ git clone https://github.com/joseantoniopau/Job-Hunt-Hacker.git
$ cd Job-Hunt-Hacker
$ ./install.sh    # installs deps, creates DB, links skill
$ ./run.sh        # serves the UI at http://127.0.0.1:8731
  1. Open the UI at http://127.0.0.1:8731 in any browser on the same machine.
  2. Click AUTOPILOT. Drop your resume, pick employment + availability, click START. Walk away.
  3. Come back to your pipeline. The skill has found, scored, tailored, and built packets for your top matches.

Other install paths: Docker, docker-compose, single binary. Optional API keys in Settings unlock richer search (SerpApi, GitHub) or smarter tailoring (Anthropic, OpenAI, local Ollama). The tool works fully without any of them.

Compliance & security

  • Local-first by default. No cloud account, no telemetry, no analytics. The vault, uploads, resumes, and packets all live on your laptop.
  • Optional bearer-token auth. Set JHH_AUTH_TOKEN if you expose the port beyond 127.0.0.1. Every /api/* request must then carry the token.
  • Per-IP rate limiting. Default 10/min on hot endpoints (search, autopilot, tailor, profile/infer). Configurable.
  • SSRF protection. URL fetchers refuse non-http(s) schemes and any host that resolves to loopback / link-local / private / multicast.
  • Upload caps + MIME validation. Default 10MB, extension whitelist enforced on every multipart endpoint.
  • Audit log retention. Daily job purges entries older than 90 days (env-configurable).
  • Pre-wipe snapshot. Before DELETE /api/data wipes anything, a full export lands in data/snapshots/; last 5 kept.
  • Auto-apply is off by default. Even when enabled, the skill PREPARES packets and queues them. Submission stays manual. Full threat model in SECURITY.md.

FAQ

Is this really free?

Yes. MIT license, runs on your laptop, no subscription, no per-seat fee. The only money you might spend is on optional LLM API keys (Anthropic / OpenAI) or SerpApi — and the tool works without any of them.

Will my data leave my computer?

Only if you configure it to. The vault is a local SQLite file. When you set an LLM API key, the contents of your evidence + job posts get sent to that provider as part of normal completion calls (use Ollama for zero cloud egress). Job source adapters make outbound requests to public job boards.

Can I share this with a friend who's job hunting?

Please do. The skill is built for individual installation per person. Their vault is their data; sharing the GitHub link is the whole adoption story.

I'm not technical. Can I use this?

If you can copy-paste four lines into Terminal, yes. The Autopilot path is one button after install. We're working on a single-binary release that skips the Python install entirely (scripts/build_binary.sh).

Does it work for non-tech roles?

Yes. 19 industry families with real keyword sets — legal, medical, finance, education, creative, hospitality, trades, science, sales, marketing, ops, support, and more. The tailoring engine, salary intelligence, and networking layer are all industry-agnostic.

Will it auto-apply to LinkedIn for me?

No. LinkedIn (and Indeed, Glassdoor) prohibit automation; we respect that. The skill prepares the application packet (tailored resume + cover letter + recruiter message + interview prep) and queues it for your review. You click the actual apply button.

How do I back up my data?

Settings → EXPORT ALL DATA produces a versioned JSON dump of every table. The wipe endpoint also writes a pre-wipe snapshot automatically; last 5 are kept. To restore: POST /api/data/snapshots/restore with the filename.

How do I uninstall?

Stop the server (Ctrl+C), delete the Job-Hunt-Hacker directory. Nothing global to remove. If you symlinked into ~/.claude/skills/, delete that symlink too. Run ./install.sh --uninstall for the automated version.

What's the architecture?

FastAPI backend, 30-table SQLite vault, 37 API routers exposing 178 endpoints across 9 job sources, vanilla brutalist HTML/CSS/JS UI, 319-test suite. Optional LLM providers (Anthropic / OpenAI / Ollama / template fallback) with full call observability. Optional Gmail + Google Calendar via OAuth. Optional Manifest V3 browser extension for evidence-grounded, never-auto-submitting autofill. Background scheduler via APScheduler. Vector retrieval via SQLite + numpy cosine (OpenAI / sentence-transformers / hashing fallback). Read the README architecture diagram.

How do I report a bug or contribute?

Open an issue or PR on GitHub. Keep the brutalist visual discipline, never weaken the no-fabrication guardrails, and document any new dependency.
