What is Job Hunt Hacker?

Job Hunt Hacker is a small, local tool that runs on your computer. It helps you look for jobs the way a careful, honest friend would: by reading what you have actually done, finding postings that match, and writing application materials that mirror the job's language only where you have evidence to back it up.

It is a single command to install, runs offline by default, and stores all of your career history in a local SQLite database that you fully control. No cloud account. No telemetry. No charge.

You can use it through Claude Code on your terminal, or through the brutalist single-page web UI it serves at http://127.0.0.1:8731.

What it does NOT do

It does not invent credentials

If you didn't claim a degree, certification, employer, title, or tool, the engine will not put it on a resume. Ever.

It does not fabricate metrics

"Improved performance by 40%" only appears if you uploaded evidence of that 40%. Otherwise it's flagged as a gap.

It does not bypass platform TOS

No login bypass, no CAPTCHA solving, no proxy rotation, no headless browser farms. Sites that require login stay manual.

It does not auto-submit on your behalf

Auto-apply is OFF by default and limited to platforms whose own API permits it. You always control submission.

It does not edit your LinkedIn

It reads pasted text or your export. It never logs in, posts, or modifies anything on LinkedIn for you.

It does not phone home

No analytics, no telemetry, no remote logging. The tool runs entirely on your machine.

Five-minute install

$ git clone https://github.com/joseantoniopau/Job-Hunt-Hacker.git
$ cd Job-Hunt-Hacker
$ ./install.sh    # installs deps, creates DB, links skill
$ ./run.sh        # serves the UI at http://127.0.0.1:8731
  1. Open the UI at http://127.0.0.1:8731 in any browser on the same machine.
  2. Click AUTOPILOT. Drop your resume, pick employment type + availability, click START. Walk away.
  3. Come back to your pipeline. The skill has found, scored, tailored, and built packets for your top matches.

Optional: add API keys in Settings for richer search (SerpAPI, GitHub) or smarter tailoring (Anthropic, OpenAI, or local Ollama). The tool works fully without any of them.

How tailoring stays honest

Every generated artifact comes with two things attached: a provenance map and an honesty report.

  • Provenance map — each bullet, paragraph, and skill listed in the output points back to one or more EvidenceSource rows in your Vault. You can audit every claim.
  • Honesty report — counts of facts used, keywords added, keywords excluded as unsupported, and explicit gaps flagged. No vague "looks good"; gaps are surfaced loudly.
  • If a job posting requires Python and you have no Python evidence, the tool refuses to add it. It suggests either uploading a project as new evidence or leaving it as a gap.
  • You can ask the assistant to be more aggressive about keyword inclusion. It will still refuse to invent dates, employers, titles, certifications, or numbers it cannot trace.

Compliance & responsibility

Scraping risk

We use the JobSpy library to fetch postings from LinkedIn, Indeed, and Glassdoor. These platforms generally prohibit automated scraping in their terms. The tool surfaces each of those sources with a GRAY badge and recommends assisted-apply only.

Auto-apply guardrails

Auto-apply is OFF by default. To enable it you must explicitly acknowledge the risk, set a daily cap, and a minimum score. It is restricted to sources whose own policies permit automation (Greenhouse, Lever, Ashby). LinkedIn, Indeed, and Glassdoor are never auto-applied to.

Kill switch

A single button or POST /api/auto-apply/halt disables everything immediately, including any scheduled runs.

FAQ

How much does this cost?

Nothing. It's MIT-licensed and runs on your laptop. If you choose to plug in an Anthropic or OpenAI API key for smarter tailoring, you pay those providers directly. Default mode uses templated fallback that costs zero.

Is my data private?

Yes. Every byte you upload — resumes, LinkedIn exports, claims, generated drafts — stays in a local SQLite database (data/jhh.db) on your machine. No analytics, no telemetry, no remote logging. The only outbound traffic happens when you explicitly call search adapters or LLM providers you configured yourself.

Can I share this with friends?

Yes, please do. It's a public skill under MIT. The whole point is that career-search tooling should not be a paid SaaS. Each person installs their own copy; nobody's data mixes.

What if I don't have any API keys?

The app runs offline with safe defaults. Templated tailoring still produces honest, evidence-grounded output — it's just less fluent than what a hosted model would generate. You can add keys later in .env and the tool picks them up on next start.

Will this edit or post to my LinkedIn?

No, never. It reads pasted text or an export file. It does not log in, post, message, follow, or modify anything on LinkedIn. The same applies to every other platform.

Can multiple people use the same install?

The tool is designed as a personal skill: one local install per person. Two people sharing a Vault would mix their evidence, which would break the honesty guarantee. Each user should clone their own copy.

How do I back up my data?

Copy the data/ and uploads/ directories. The Settings page also has a one-click "Export all data" button that bundles your Vault, resumes, and applications into a JSON archive.

How do I uninstall?

Stop the server (Ctrl+C in the terminal where ./run.sh is running), then delete the Job-Hunt-Hacker directory. There's nothing global to remove. If you symlinked it into ~/.claude/skills/, remove that symlink too.
